# 抑制策略配置推送指南

## ? 配置文件说明

在 `backend/config.py` 中，现在有4个独立的SSH配置，分别对应不同的抑制策略：

### 1. TCP RST攻击配置 (`TCP_RST_CONFIG`)

```python
TCP_RST_CONFIG = {
    'host': '192.168.1.100',      # 中继节点IP地址
    'port': 22,                    # SSH端口
    'username': 'relay',           # SSH用户名
    'password': 'relay123',        # SSH密码
    'share_path': '/opt/relay_share',  # 共享目录路径
    'poll_interval': 2,            # 轮询间隔（秒）
    'connection_timeout': 10,      # 连接超时（秒）
    'enabled': True                # 是否启用
}
```

**用途**: TCP RST攻击通过中继节点执行，需要配置中继节点的SSH信息

---

### 2. IP黑名单配置 (`IP_BLACKLIST_CONFIG`)

```python
IP_BLACKLIST_CONFIG = {
    'host': '192.168.1.200',             # 网关设备IP
    'port': 22,
    'username': 'gateway',
    'password': 'gateway123',
    'config_path': '/opt/suppression_config',  # 远程配置目录
    'connection_timeout': 10,
    'enabled': True
}
```

**用途**: 将IP黑名单推送到网关设备，设备通过iptables ipset屏蔽这些IP

---

### 3. 域名黑名单配置 (`DOMAIN_BLACKLIST_CONFIG`)

```python
DOMAIN_BLACKLIST_CONFIG = {
    'host': '192.168.1.201',             # DNS服务器IP
    'port': 22,
    'username': 'dns',
    'password': 'dns123',
    'config_path': '/opt/suppression_config',
    'connection_timeout': 10,
    'enabled': True
}
```

**用途**: 将域名黑名单推送到DNS服务器，通过dnsmasq将这些域名解析到0.0.0.0

---

### 4. 丢包策略配置 (`PACKET_LOSS_CONFIG`)

```python
PACKET_LOSS_CONFIG = {
    'host': '192.168.1.200',             # 网关设备IP（可与IP黑名单同一设备）
    'port': 22,
    'username': 'gateway',
    'password': 'gateway123',
    'config_path': '/opt/suppression_config',
    'connection_timeout': 10,
    'enabled': True
}
```

**用途**: 将丢包策略推送到网关设备，通过iptables实现概率性丢包

---

## ? 推送内容详细说明

### 1. ? 推送IP黑名单到网关设备

**点击按钮**: "? 推送到网关设备" （在IP黑名单Tab）

**推送内容**:
```
数据来源: 数据库表 ip_blacklist
推送到: IP_BLACKLIST_CONFIG['host']
文件路径: /opt/suppression_config/ip_blacklist.txt
文件格式: 纯文本，每行一个IP地址
```

**文件示例**:
```text
# IP Blacklist - Auto-generated by platform
# Total IPs: 3
192.168.1.100
10.0.0.50
172.16.0.10
```

**远程设备行为**:
- 网关设备上的 `gateway_black_white_file.py` 监控此文件
- 文件更新时，自动将IP加入iptables ipset集合
- 匹配这些IP的流量会被DROP

---

### 2. ? 推送域名黑名单到DNS服务器

**点击按钮**: "? 推送到DNS服务器" （在域名黑名单Tab）

**推送内容**:
```
数据来源: 数据库表 domain_blacklist
推送到: DOMAIN_BLACKLIST_CONFIG['host']
文件路径: /opt/suppression_config/domain_blacklist.txt
文件格式: 纯文本，每行一个域名
```

**文件示例**:
```text
# Domain Blacklist - Auto-generated by platform
# Total Domains: 3
malware.com
botnet.net
c2server.org
```

**远程设备行为**:
- DNS服务器上的 `dns_black_white_file.py` 监控此文件
- 文件更新时，自动生成dnsmasq配置 `/etc/dnsmasq.d/dns_blacklist.conf`
- 将这些域名解析到0.0.0.0，实现DNS劫持屏蔽
- 自动重启dnsmasq服务

---

### 3. ? 推送丢包策略到网关设备

**点击按钮**: "? 推送到网关设备" （在丢包策略Tab）

**推送内容**:
```
数据来源: 数据库表 packet_loss_policy (仅enabled=TRUE的记录)
推送到: PACKET_LOSS_CONFIG['host']
文件路径: /opt/suppression_config/packet_loss.json
文件格式: JSON格式，key为IP，value为丢包率(0-1)
```

**文件示例**:
```json
{
  "192.168.1.100": 0.3,
  "10.0.0.50": 0.5,
  "172.16.0.10": 0.2
}
```

**说明**:
- `0.3` = 30%丢包率
- `0.5` = 50%丢包率
- `0.2` = 20%丢包率

**远程设备行为**:
- 网关设备上的 `gateway_packet_loss_file.py` 监控此文件
- 文件更新时，自动更新iptables规则
- 使用 `-m statistic --mode random --probability` 实现概率性丢包
- 同时对进入和发出的流量生效

---

### 4. ? TCP RST攻击

**点击按钮**: "启动TCP RST攻击" （在TCP RST攻击Tab）

**推送内容**:
```
数据来源: 前端表单（目标IP、端口、网络接口）
推送到: TCP_RST_CONFIG['host']
文件路径: /opt/relay_share/commands/attack_<attack_id>.json
文件格式: JSON格式，包含攻击参数
```

**文件示例**:
```json
{
  "attack_id": "attack_1747022800_abcd1234",
  "target_ip": "192.168.1.10",
  "target_port": 80,
  "capture_interface": "eth0",
  "inject_interface": "eth1",
  "timestamp": "2024-01-10 10:00:00"
}
```

**远程设备行为**:
- 中继节点上的 `relay_node_file_monitor.py` 监控此文件
- 检测到新的命令文件后，调用 `tcp_rst.py` 脚本
- 监听目标IP和端口的TCP流量
- 伪造RST包中断TCP连接
- 将攻击状态写入 `/opt/relay_share/status/` 目录
- 平台通过轮询读取状态文件获取实时连接信息

## ? 使用流程

### 步骤1: 配置SSH连接信息

编辑 `backend/config.py`，根据实际情况填写各个设备的SSH信息。

### 步骤2: 启动平台服务

```bash
cd /home/spider/31339752/backend
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000
```

查看启动日志，确认各个管理器初始化成功：
```
? TCP RST攻击中继文件管理器已初始化 -> 192.168.1.100
? IP黑名单推送管理器已初始化 -> 192.168.1.200
? 域名黑名单推送管理器已初始化 -> 192.168.1.201
? 丢包策略推送管理器已初始化 -> 192.168.1.200
```

### 步骤3: 部署远程设备脚本

参考 `SUPPRESSION_STRATEGIES_FILE_DEPLOYMENT.md` 部署对应的监控脚本。

### 步骤4: 在平台界面推送配置

1. 登录平台大屏
2. 进入 **抑制阻断策略** 页面
3. 在对应的Tab中添加策略
4. 点击对应的 **? 推送** 按钮

### 步骤5: 验证

检查远程设备日志，确认配置已应用：

```bash
# IP黑名单
sudo journalctl -u ip-blacklist -f

# 域名黑名单
sudo journalctl -u dns-blacklist -f

# 丢包策略
sudo journalctl -u packet-loss -f

# TCP RST攻击
sudo journalctl -u relay-monitor -f
```

## ? 总结

### 关键要点

1. **独立配置**: 每个策略有自己的SSH配置，可以推送到不同的服务器
2. **自动应用**: 远程设备监控配置文件，自动应用更改
3. **实时生效**: 文件更新后立即触发iptables/dnsmasq规则更新
4. **安全推送**: 通过SSH加密传输，平台IP不暴露

### 推送内容总结

| 策略 | 推送内容 | 文件格式 | 目标设备 |
|------|---------|---------|---------|
| IP黑名单 | IP地址列表 | 纯文本 | 网关设备 |
| 域名黑名单 | 域名列表 | 纯文本 | DNS服务器 |
| 丢包策略 | IP+丢包率 | JSON | 网关设备 |
| TCP RST攻击 | 攻击参数 | JSON | 中继节点 |

祝使用愉快！?
