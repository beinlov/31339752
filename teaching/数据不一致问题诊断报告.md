## ✅ 哪个数据才是真实数据？

### 正确答案：**图二和图三的 116,108 是更准确的数据**

**理由**：

1. ✅ **逻辑正确**: node.py 的计算方式 `china_botnet + global_botnet(非中国)` 避免了重复计算中国节点

2. ✅ **语义清晰**: 
   - `china_botnet_ramnit` 专门存储中国节点数据
   - `global_botnet_ramnit` 存储全球所有国家的节点数据
   - 正确的总数 = 中国节点(单独统计) + 其他国家节点

3. ❌ **amount.py 的问题**: 直接返回 `global_botnet` 表总和会造成语义混淆
   - 如果 `global_amount` 表示"全球总数"，应该等于 china + 其他国家
   - 如果 `global_amount` 表示"global_botnet 表的数据"，那么名称会误导用户
   - 当前实现中，`global_botnet` 已包含中国节点，但 `china_botnet` 也有中国节点，容易造成理解偏差

### 为什么 amount.py 返回 116,090？

因为 `global_botnet_ramnit` 表中的中国节点是 27,888 个（比 china_botnet 少 18 个）:

```
global_amount = 中国(27,888) + 非中国(88,202) = 116,090
```

这说明 **`global_botnet` 表中的数据不完整**，缺少了 18 个中国节点。

## 🔧 修复方案

### 方案1：修复数据同步问题（推荐）

**根本解决方案**：确保 `china_botnet` 和 `global_botnet` 表中的中国节点数据一致

**步骤**：
1. 检查数据聚合器的逻辑，确保同步机制正常
2. 找出缺失的 18 个中国节点，补充到 `global_botnet` 表
3. 或者删除 `global_botnet` 中的中国数据，重新从源数据聚合

**优点**：
- 从根本上解决数据不一致问题
- 所有接口的数据将保持一致

**实施代码**：

```python
# 检查数据同步脚本
import pymysql
from config import DB_CONFIG

conn = pymysql.connect(**DB_CONFIG)
cursor = conn.cursor()

# 对所有僵尸网络类型进行检查
cursor.execute("SELECT name FROM botnet_types")
botnets = cursor.fetchall()

for (botnet_name,) in botnets:
    cursor.execute(f"SELECT SUM(infected_num) FROM china_botnet_{botnet_name}")
    china_total = cursor.fetchone()[0] or 0
    
    cursor.execute(f"SELECT SUM(infected_num) FROM global_botnet_{botnet_name} WHERE country = '中国'")
    china_in_global = cursor.fetchone()[0] or 0
    
    if china_total != china_in_global:
        print(f"⚠️  {botnet_name}: china表={china_total}, global中国={china_in_global}, 差异={china_total - china_in_global}")
```

### 方案2：修改 amount.py 的计算逻辑

**目标**：让 `amount.py` 的 `global_amount` 与 `node.py` 的 `total_nodes` 保持一致

**修改文件**: `/backend/router/amount.py`

**修改内容**:

```python
# 原代码（第48-54行）
global_query = f"""
    SELECT COALESCE(SUM(infected_num), 0) as global_total
    FROM global_botnet_{botnet_name}
"""
cursor.execute(global_query)
global_total = cursor.fetchone()['global_total']

# 修改为（推荐）
# 方式1: 计算真正的全球总数（china + 非中国）
china_query = f"""
    SELECT COALESCE(SUM(infected_num), 0) as china_total
    FROM china_botnet_{botnet_name}
"""
cursor.execute(china_query)
china_total = cursor.fetchone()['china_total']

global_non_china_query = f"""
    SELECT COALESCE(SUM(infected_num), 0) as global_non_china
    FROM global_botnet_{botnet_name}
    WHERE country != '中国'
"""
cursor.execute(global_non_china_query)
global_non_china = cursor.fetchone()['global_non_china']

# 全球总数 = 中国 + 非中国
global_total = china_total + global_non_china

response_data.append({
    "name": botnet_name,
    "china_amount": int(china_total),
    "global_amount": int(global_total)  # 现在是真正的全球总数
})
```

**优点**：
- 确保前后端数据一致
- 语义更清晰："global_amount" 真正表示全球总数
