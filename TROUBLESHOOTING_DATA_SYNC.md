# C2端与平台端数据同步故障排查

## 🔍 问题现象

**症状：** C2端显示有缓存数据（101条），但平台端拉取返回0条数据

**日志分析：**

### C2端日志：
```
当前缓存: 101 条
拉取请求: 返回 0 条记录（未确认，等待服务器确认）
```

### 平台端日志：
```
使用序列ID断点续传: since_seq=396
没有新数据（返回0条）
```

---

## 🎯 根本原因

**C2端缓存的101条数据的 `seq_id` 都 ≤ 396，已经被平台拉取过了！**

### 数据流程说明：

```
┌─────────────────────────────────────────────────────────────┐
│ 正常流程：                                                   │
│                                                              │
│ 1. C2生成日志 → 写入缓存 (seq_id=1)                         │
│ 2. 平台拉取 (since_seq=0) → 获取 seq_id=1 的数据            │
│ 3. 平台处理成功 → 发送确认                                   │
│ 4. C2收到确认 → 标记 pulled=1                                │
│ 5. 平台保存游标 (last_seq_id=1)                              │
│                                                              │
│ 下次拉取：                                                    │
│ 6. 平台拉取 (since_seq=1) → 只获取 seq_id > 1 的数据        │
│ 7. 如果没有新数据 → 返回0条 ✅ 这是正常的！                 │
└─────────────────────────────────────────────────────────────┘
```

### 步骤2：根据诊断结果判断

#### 情况A：缓存中没有 seq_id > 396 的记录 ✅ 正常

**解决方法：**
1. **等待自然产生数据**（推荐）
   - C2收到新的节点上线 → 生成日志 → 平台自动拉取
   - C2执行节点清除 → 生成日志 → 平台自动拉取

2. **手动触发数据生成**（测试用）


2. C2端API返回数据异常
3. 网络问题

**解决方法：**

## 🛠️ 常见问题和解决方案

### Q1: 为什么缓存一直是101条，不增加也不减少？

**A:** 可能的原因：

1. **C2没有新的日志生成**
   - 检查：没有新节点上线，没有清除操作
   - 解决：等待或手动触发

2. **C2日志读取器停止工作**
   - 检查：查看C2日志，看是否有 "📖 开始读取" 的输出
   - 解决：重启C2数据服务器

3. **缓存的101条数据都已被拉取（pulled=1）**
   - 检查：运行 `check_cache_status.py`
   - 解决：这是正常的，等待新数据

---

### Q2: 平台一直显示 "since_seq=396"，从不更新？

**A:** 可能的原因：

1. **C2返回0条数据**
   - 检查：C2没有 seq_id > 396 的未拉取数据
   - 解决：等待新数据或重置状态

2. **平台的断点续传文件被锁定**
   - 检查：`.remote_puller_state.json` 文件权限
   - 解决：删除该文件，重启日志处理器

---

### Q3: 如何强制平台重新拉取所有数据？

**A:** 

```bash
# 方法1：重置平台断点续传状态
cd d:\workspace\botnet\backend\log_processor
python reset_pull_state.py

# 方法2：直接删除状态文件
rm .remote_puller_state.json

# 方法3：重置C2端缓存（慎用！会清空所有缓存）
# 在C2服务器上
cd /path/to/c2_server
rm data_cache.db
# 重启C2数据服务器
```

---

### Q4: 平台重启后，会重复拉取数据吗？

**A:** 不会！

- 平台使用 `.remote_puller_state.json` 保存断点续传状态
- 重启后会从 `last_seq_id` 继续拉取
- 数据库会自动去重（基于IP+timestamp）

---

### Q5: C2端缓存会无限增长吗？

**A:** 不会！

C2端有自动清理机制：
- 已拉取的数据（pulled=1）会在一定时间后删除
- 或者达到缓存上限后删除旧数据

检查清理配置：
```python
# backend/remote/config.json
{
  "cache": {
    "max_cache_size": 10000,
    "retention_hours": 24,
    "cleanup_interval": 3600
  }
}
```
