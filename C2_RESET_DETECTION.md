# C2端重置自动检测机制

## 🎯 问题场景

当C2端清空日志/缓存后，`seq_id` 会从1重新开始计数，但平台端保存的断点续传游标仍是旧值（如396），导致永远拉取不到数据。

### 场景示例：

```
时间线：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
昨天 20:00
  C2端缓存: seq_id 1-396
  平台拉取: since_seq=0 → 获取全部
  平台保存: last_seq_id=396 ✅

今天 08:00 (运维操作)
  C2端清空日志/缓存 
  → seq_id 重新从1开始 🔄
  → 新数据: seq_id 1-100

今天 08:10 (问题出现)
  平台拉取: since_seq=396
  C2查询: WHERE seq_id > 396
  → 返回 0 条 ❌ 问题！
  
结果：永远拉取不到新数据！
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## ✅ 解决方案：三层防护

### 第1层：C2端返回全局最大seq_id

**实现位置：** `backend/remote/c2_data_server.py`

```python
# 在handle_pull中添加
global_max_seq_id = 缓存中的最大seq_id

response = {
    'max_seq_id': 本次返回的最大seq_id,
    'global_max_seq_id': 缓存中的全局最大seq_id,  # 🆕 新增
    ...
}
```

**作用：** 让平台知道C2端当前的最大seq_id

---

### 第2层：平台端自动检测倒退

**实现位置：** `backend/log_processor/remote_puller.py`

```python
# 在_pull_from_c2中添加检测逻辑
if 平台的since_seq > C2的global_max_seq_id:
    logger.warning("检测到C2端重置！")
    # 自动清除断点续传状态
    del self.last_seq_ids[name]
    self.save_state()
    logger.info("已重置断点续传，下次将从头拉取")
```

**触发条件：**
- 平台游标：`since_seq=396`
- C2返回：`global_max_seq_id=100`
- 判断：`396 > 100` → 检测到重置！

**自动操作：**
1. 清除平台的断点续传状态
2. 下次拉取时不带`since_seq`参数
3. 从头拉取所有数据

---

### 第3层：手动重置工具（备用）

**工具位置：** `backend/log_processor/reset_pull_state.py`

```bash
# 手动重置断点续传状态
python reset_pull_state.py "C2-utg_q_008-remote"
```

**使用场景：**
- 自动检测失败时
- 需要强制重新拉取时
- 故障排查时

## 📋 常见问题

### Q1: 自动检测会误判吗？

**A:** 不会！只有在**明确的倒退**时才触发：
- 触发条件：`平台游标 > C2全局最大seq_id`
- 如果是正常的"没有新数据"，C2的`global_max_seq_id`会等于或大于平台游标
- 只有C2重置后，才会出现倒退

---

### Q2: 如果C2端使用全局递增seq_id（永不重置），会影响吗？

**A:** 不会！
- 如果C2的seq_id永不重置，`global_max_seq_id`会持续递增
- 检测条件永远不会触发
- 系统正常工作

---

### Q3: 重置后会重复拉取数据吗？

**A:** 会重新拉取，但数据库会自动去重：
- 数据库使用 `(ip, timestamp)` 作为唯一索引
- 重复数据会被忽略，不会重复入库
- 日志会显示 "重复: X 条"

---

### Q4: 自动检测的延迟是多少？

**A:** 最多一个拉取周期（10秒）：
- 第1次拉取：检测到重置，清除状态
- 第2次拉取：从头拉取，恢复正常
- 总延迟：10-20秒

---

### Q5: 如果网络问题导致拉取失败，会误触发重置吗？

**A:** 不会！
- 网络失败时，平台不会收到响应
- 检测逻辑只在成功拉取后执行
- `global_max_seq_id`为None时不触发检测

## 🎯 总结

**问题：** C2端清空缓存后，seq_id重置，平台无法拉取数据

**方案：** 三层防护机制
1. C2返回全局最大seq_id
2. 平台自动检测倒退并重置
3. 手动重置工具备用

**结果：**
- ✅ 自动检测，无需人工干预
- ✅ 10秒内自动恢复
- ✅ 安全可靠，不会误判
- ✅ 完整的日志和监控
