# Industry Distribution Country Filter Fix

## Problem

**Issue**: Industry distribution showed 989 nodes, but China active nodes only 964

**Root Cause**: The industry distribution API did not filter by country, including all countries' data.

## Analysis

### Data Breakdown
- **Total active nodes**: 989
  - China (жаЙњ): 964
  - USA (УРЙњ): 17
  - Japan (ШеБО): 5
  - Germany (ЕТЙњ): 1
  - Peru (УиТГ): 1
  - Netherlands (КЩРМ): 1

### Before Fix
```sql
SELECT industry, COUNT(*) 
FROM botnet_nodes_utg_q_008
WHERE status = 'active'
GROUP BY industry
```
**Result**: 989 nodes (all countries)

### After Fix
```sql
SELECT industry, COUNT(*) 
FROM botnet_nodes_utg_q_008
WHERE status = 'active'
  AND country = 'жаЙњ'
GROUP BY industry
```
**Result**: 964 nodes (China only)

## Solution

**File**: `backend/router/amount.py`
**Line**: 203-204

### Code Change

```python
# Before
WHERE status = '{safe_display_mode}'

# After
WHERE status = '{safe_display_mode}'
    AND country = 'жаЙњ'
```

## Impact

### Dashboard Display
- **Before**: Industry total = 989 (mismatched with China active 964)
- **After**: Industry total = 964 (matches China active nodes)

### API Response
```json
{
  "name": "ЪЕЭЅПэДј",
  "value": 450,
  "percentage": 46.7
}
```

Now only includes Chinese nodes in the calculation.
