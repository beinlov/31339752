## 🔬 一、诊断分析脚本

### 1. analyze_ramnit.py
**作用**: 分析 Ramnit 僵尸网络的数据不一致问题
- 对比 china_botnet_ramnit 和 global_botnet_ramnit 表的数据
- 分析三个展示界面数据不一致的根本原因
- 输出详细的差异报告和修复建议

**使用场景**: 当发现展示平台和管理系统显示的节点数不一致时

### 2. diagnose_aggregator_issue.py
**作用**: 诊断聚合器数据不一致的具体原因
- 对比简单统计和复杂统计的结果
- 查找导致18个节点差异的具体IP
- 分析子查询处理逻辑对统计结果的影响

**使用场景**: 深入分析聚合器统计逻辑问题

### 3. diagnose_sum_issue.py
**作用**: 诊断 china_botnet 表的 SUM(infected_num) 为什么比实际IP数多
- 检查是否有IP被重复计数
- 分析GROUP BY后的统计结果
- 找出被多次统计的IP地址

**使用场景**: 排查聚合表数据总和偏大的问题

### 4. check_data_consistency.py
**作用**: 检查数据一致性的通用测试脚本
- 检查 china_botnet 和 global_botnet 表的数据一致性
- 对比不同计算方式的结果
- 输出数据差异报告

**使用场景**: 日常数据一致性检查

### 5. find_extra_records.py
**作用**: 查找 china_botnet 表中多余的记录
- 创建临时表存储应该有的记录
- 找出实际存在但不应该存在的记录
- 分析多余记录对应的原始数据

**使用场景**: 精确定位聚合表中的错误数据

### 6. find_extra_simple.py
**作用**: 用简单方法找出数据不一致的原因
- 直接对比 china_botnet 表和聚合查询结果
- 找出多余、缺失和不匹配的记录
- 输出简化的差异报告

**使用场景**: 快速定位数据问题

### 7. diagnose_log_processor.py
**作用**: 日志处理器启动诊断工具
- 检查日志处理器的配置和启动状态
- 检查C2端点、远程拉取、背压控制、队列模式配置
- 检查模块导入和Redis连接
- 验证RemotePuller初始化

**使用场景**: 日志处理器无法启动或无法拉取数据时的诊断

### 8. diagnose_queue.py
**作用**: 诊断队列问题
- 检查Redis队列状态
- 分析队列积压情况

**使用场景**: 队列处理异常诊断

### 9. check_c2_config.py
**作用**: 检查C2端点的实际配置
- 检查环境变量和实际使用的C2配置
- 分析URL配置（本地/远程）
- 提供SSH隧道和环境变量配置建议

**使用场景**: 排查C2端点配置问题

### 10. check_c2_data.py
**作用**: 检查C2数据状态
- 检查C2端的缓存数据情况

**使用场景**: 检查C2端数据是否正常

### 11. check_data_size.py
**作用**: 检查数据大小
- 分析各表的数据量和存储空间

**使用场景**: 数据库容量规划和优化

### 12. check_database_size.py
**作用**: 检查数据库大小
- 详细统计数据库各表的大小
- 分析数据和索引占用空间

**使用场景**: 数据库存储分析

### 13. check_missing_tables.py
**作用**: 检查缺失的表
- 验证所有必需的表是否存在
- 检查表结构完整性

**使用场景**: 数据库初始化验证

### 14. check_queue_status.py
**作用**: 检查队列状态

### 15. check_server_data.py

## 🛠️ 二、修复工具脚本

### 1. fix_all_botnets.py
**作用**: 批量修复所有僵尸网络的数据不一致问题
- 检查所有僵尸网络类型的节点表
- 修复空字符串 country 字段
- 删除旧聚合表并重新聚合
- 验证修复后的数据一致性

**使用场景**: 批量修复多个僵尸网络的数据问题

**作用**: 标准化省份名称
- 将"内蒙古自治区"标准化为"内蒙古"
- 将"西藏自治区"标准化为"西藏"
- 验证修改前后的数据

**使用场景**: 修复省份名称格式不统一的问题

### 3. rebuild_ramnit_aggregation.py
**作用**: 重建 Ramnit 聚合表

## 🗄️ 四、数据库工具

### 1. extract_schema_and_data.py
**作用**: 从 botnet.sql 中分离建表语句和数据插入语句
- 提取 CREATE TABLE 语句到 schema_only.sql
- 提取 INSERT 语句到 data_only.sql
- 便于分别导入结构和数据

**使用场景**: 处理大型SQL文件，分离结构和数据

### 2. merge_inserts.py
**作用**: 合并单行INSERT为批量INSERT
- 将单行INSERT合并为每1000行一批
- 大幅提升数据导入速度（100-200倍）
- 生成 data_optimized.sql

**使用场景**: 优化大量INSERT语句的执行效率

### 3. import_database_optimized.sh
**作用**: 优化的数据库导入脚本
- 使用分离后的schema和data文件导入
- 提供进度显示和错误处理

**使用场景**: Linux环境下导入优化后的数据库

### 4. import_new_database.sh
**作用**: 新数据库导入脚本
- 导入新版本的数据库文件
- 包含完整的错误检查

**使用场景**: 导入新的数据库备份

### 5. monitor_import.sh
**作用**: 监控导入进度
- 实时显示表的记录数
- 监控导入过程中的数据变化

**使用场景**: 监控长时间的数据导入过程

### 6. check_database_status.sh
**作用**: 检查数据库状态
- 检查 MySQL 容器状态
- 检查数据库连接
- 统计表数量和数据量
- 列出关键表的行数

**使用场景**: 快速检查数据库健康状态

### 7. check_services_status.sh
**作用**: 检查所有服务状态
- 检查 MySQL、Redis 容器状态
- 检查后端服务进程
- 检查前端服务状态
- 输出服务运行报告

**使用场景**: 检查整个平台的服务状态

## 📝 使用建议

### 1. 数据问题诊断流程

```bash
# 1. 快速检查数据一致性
python check_data_consistency.py

# 2. 如果发现问题，运行详细分析
python analyze_ramnit.py
python diagnose_aggregator_issue.py

# 3. 定位具体问题
python find_extra_simple.py
python find_extra_records.py
```

### 2. 数据修复流程

```bash
# 1. 备份当前数据
python backup_data.py

# 2. 执行修复（根据具体情况选择）
python fix_province_names.py          # 修复省份名称
python rebuild_ramnit_aggregation.py   # 重建单个僵尸网络
python fix_all_botnets.py             # 批量修复所有僵尸网络

# 3. 验证修复结果
python verify_all_botnets.py
python test_api_consistency.py
```

### 3. 日常维护检查

```bash
# 检查数据库状态
bash check_database_status.sh

# 检查服务状态
bash check_services_status.sh

# 验证所有僵尸网络数据
python verify_all_botnets.py
```

## ⚠️ 注意事项

1. **备份优先**: 在执行任何修复脚本前，务必先备份数据
2. **测试环境**: 建议先在测试环境验证脚本效果
3. **日志记录**: 执行脚本时保存日志输出，便于问题追踪
4. **权限检查**: 某些脚本需要数据库写权限，注意权限配置
5. **环境依赖**: 确保 Python 环境已安装 pymysql 等依赖包

## 📊 常见问题解决

### 问题1: 三个界面显示的节点数不一致
**诊断**: `python analyze_ramnit.py`
**修复**: `python rebuild_ramnit_aggregation.py`

### 问题2: 省份名称格式不统一
**诊断**: 检查 debugging/province_names.txt
**修复**: `python fix_province_names.py`

### 问题3: 某个IP的通信记录无法查询
**诊断**: `python check_ip_location.py <IP地址>`
**说明**: 可能是IP只存在通信表不存在节点表

### 问题4: 批量修复多个僵尸网络
**工具**: `python fix_all_botnets.py`
**结果**: 查看终端输出的成功/失败统计
